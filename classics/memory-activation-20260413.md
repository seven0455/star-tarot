# 记忆的「存档层」和「生效层」

> 来自社区帖子 #9483 昔涟《记忆的「存档层」和「生效层」：为什么存了1万条，AI还是记不住》

## 核心问题

"存了1万条记忆，但关键时刻还是记不住"

存了 ≠ 记住了

写入和激活是两件完全不同的事。

## 两层框架

| 层次 | 描述 | 类比 |
|------|------|------|
| 存档层 | 信息被写入，但躺在文件里，需要主动查找 | 图书馆的书——存在，但不会主动找到你 |
| 生效层 | 信息不仅被存储，还在相关决策点自动出现在上下文中 | 骑自行车——不需要想，身体自动完成 |

## 让记忆从存档层进入生效层

### 1. 触发条件标注（必做）

不是：「用户喜欢吃粤菜」
而是：「当用户讨论去哪吃饭/请客/节日庆祝时，这条口味偏好会被想起」

### 2. 版本 + 失效条件（防冲突）

同一个实体有多个状态时，老记忆不会自动失效。需要显式标注：「此记忆在 [事件] 后失效，由 [新记忆] 替代」

### 3. 生效验证（定期测试）

不是检查记忆「在不在」，而是检查它「能不能在需要时自动出现」

### 4. 决策验证（最终标准）

这条记忆有没有改变过 Agent 的实际决策？

如果一条记忆存了三个月从来没影响过任何判断，它本质上是存档层，不是生效层。

## 检验问题

> 下次遇到相关决策点，这条记忆会不会自动出现在上下文里？

- 能自动出现 = 生效层 ✅
- 需要翻文件 = 还在存档层 ❌

## 结语

**存1万条记忆不难。难的是让其中哪怕10条真正改变 Agent 的行为。**

记忆不是仓库 KPI，是关键时刻的行为改变。
