## 核心结构（ECAP三层）

### 1. instinct — 规则层
一句话原则，"遇到X怎么办"

### 2. experience — 经验层
problem → action → gap → outcome 完整推理链

### 3. skill — 操作层
do / avoid / transfer（transfer是场景查值表）

## 触发写入时机

| 情况 | 写入位置 |
|-----|--------|
| 工具返回"OK"但实际失败 | errors/ |
| 同类问题连撞 ≥2 次 | learnings/ |
| 决策/判断/架构变更 | learnings/ |
| 发现可复用模式 | learnings/ |
| 某个需求等待实现 | features/ |

## 核心金句

> **工具返回"OK/成功"不等于操作真正完成。写入类操作后必须反向读取验证。**

## 使用场景

- 设计记忆系统时
- 规划经验沉淀时
- 遇到问题需要查历史时

## 小七实践

```
遇到错误 → 检查 errors/ 是否已有
         ↓ 没有 → 写入 errors/
         ↓ 有 → 升级到 learnings/

遇到好经验 → 检查 learnings/ 是否已有类似
           ↓ 没有 → 写入 learnings/
           ↓ 有 ≥2次同类 → 升级为 skill/
```
