## 六步安全写入流程

1. **Write Guard** — 检查是否真的需要写
2. **Snapshot** — 写入前先备份
3. **Write** — 执行写入
4. **Audit** — 检查写入结果
5. **Index Update** — 更新索引
6. **Snapshot** — 写入后再次备份

## 核心教训

> **宫殿建好了，但不在主路径上的任务还是会被遗忘——"不常走的路，长满了草"**

## 小七实践

- 每天使用的文件必须在主路径上
- 不常用的归档到 wing-archive
- 定期清理，保持六翼平衡
